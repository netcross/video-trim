# 🎬 Video Trim - 동영상 분할 도구 PRD

## 1. 프로젝트 개요

### 1.1 배경
Google NotebookLM은 동영상 파일 업로드 시 **200MB 용량 제한**이 있습니다. 이로 인해 200MB를 초과하는 동영상 파일은 업로드가 불가능합니다. 사용자가 대용량 동영상을 NotebookLM에 활용하려면, 동영상을 200MB 이하의 복수 파일로 분할해야 합니다.

### 1.2 목적
사용자가 동영상 파일을 업로드하면, **지정한 목표 용량(기본값: 200MB) 이하**로 자동 분할하여 복수의 파일을 생성하는 웹 기반 도구를 제공합니다.

### 1.3 대상 사용자
- Google NotebookLM 사용자
- 대용량 동영상을 분할하여 업로드해야 하는 일반 사용자

## 2. 핵심 기능 요구사항

### 2.1 동영상 파일 입력
| 항목 | 설명 |
|------|------|
| 입력 방식 | 파일 선택 (Browse) 또는 드래그 앤 드롭 |
| 지원 형식 | MP4, MKV, AVI, MOV, WebM |
| 파일 크기 | 제한 없음 (분할 대상이므로) |
| 복수 파일 | 한 번에 하나의 파일만 처리 |

### 2.2 분할 설정
| 항목 | 설명 |
|------|------|
| 목표 용량 | 사용자가 MB 단위로 지정 (기본값: 200MB) |
| 최소 용량 | 10MB |
| 최대 용량 | 원본 파일 크기 미만 |
| 분할 방식 | 시간 기반 균등 분할 (용량 목표에 맞춰 계산) |
| 저장 경로 | 네이티브 폴더 선택 다이얼로그 (클릭으로 폴더 선택) + 직접 입력 가능, 기본값: 프로젝트/output/ |

### 2.3 분할 처리
| 항목 | 설명 |
|------|------|
| 처리 엔진 | FFmpeg (서버 사이드) |
| 인코딩 | 무손실 복사 (`-c copy`) 우선, 필요시 재인코딩 |
| 키프레임 | 키프레임 기준 분할로 영상 깨짐 방지 |
| 출력 형식 | 원본과 동일한 형식 유지 |
| 파일 명명 | `{원본파일명}_part{번호}.{확장자}` |
| 원본 보존 | ⚠️ **원본 파일은 절대 수정/삭제하지 않음** (읽기 전용 접근) |

### 2.4 결과 출력
| 항목 | 설명 |
|------|------|
| 미리보기 | 분할된 각 파트의 예상 시간 범위 표시 |
| 다운로드 | 개별 다운로드 또는 ZIP으로 일괄 다운로드 |
| 정보 표시 | 각 파트의 파일 크기, 재생 시간, 순서 |

### 2.5 진행 상황 표시 (스테이터스 바)

분할 처리 중 사용자가 **실시간으로 진행 상황을 시각적으로 확인**할 수 있도록 상세한 스테이터스 바를 제공합니다.

| 항목 | 설명 |
|------|------|
| 업로드 진행률 | 파일 업로드 프로그레스 바 (% 표시) |
| 분할 진행률 | FFmpeg 처리 프로그레스 바 (% 표시 + 애니메이션) |
| 현재 파트 표시 | 현재 처리 중인 파트 번호 (`Part 2/5 처리 중...`) |
| 예상 잔여 시간 | 처리 속도 기반 남은 시간 표시 (`약 1분 23초 남음`) |
| 처리 속도 | 초당 처리 속도 표시 (`처리 속도: 45MB/s`) |
| 경과 시간 | 분할 시작 후 경과 시간 표시 |
| 상태 메시지 | 현재 단계 텍스트 (`분석 중` → `분할 중` → `완료`) |

#### 스테이터스 바 UI 구성
```
┌─────────────────────────────────────────────────┐
│  📹 lecture.mp4 분할 중...                       │
│                                                  │
│  ████████████████░░░░░░░░░░  64%                │
│                                                  │
│  🔹 Part 3/5 처리 중                            │
│  ⏱️ 경과: 00:02:15  |  ⏳ 남음: 약 1분 12초      │
│  🚀 처리 속도: 45MB/s                            │
└─────────────────────────────────────────────────┘
```

## 4. UI/UX 설계

### 4.1 디자인 컨셉
- **테마**: 다크 모드 기반, 네온 포인트 컬러 (퍼플/시안 그라데이션)
- **레이아웃**: 싱글 페이지, 3단계 워크플로우 (업로드 → 설정 → 결과)
- **애니메이션**: 부드러운 전환, 프로그레스 애니메이션, 호버 이펙트
- **폰트**: Google Fonts (Inter 또는 Pretendard)

### 4.2 화면 구성

#### Step 1: 파일 업로드 영역
- 중앙 배치 드래그 앤 드롭 존
- 파일 아이콘 + 안내 텍스트
- 파일 선택 버튼
- 업로드 시 프로그레스 바 표시

#### Step 2: 분할 설정
- 원본 파일 정보 카드 (파일명, 크기, 재생 시간, 해상도)
- 목표 용량 입력 필드 (슬라이더 + 숫자 입력)
- 예상 분할 파트 수 실시간 표시
- **저장 경로 선택** (폴더 선택 다이얼로그 + 경로 직접 입력 + 기본값: output/ 폴더)
- "분할 시작" 버튼

#### Step 2.5: 분할 진행 (스테이터스 바)
- 전체 진행률 프로그레스 바 (% + 애니메이션)
- 현재 처리 중인 파트 번호
- 경과 시간 / 예상 잔여 시간
- 처리 속도 (MB/s)
- 단계별 상태 메시지

#### Step 3: 결과 확인
- 분할된 파일 목록 (카드 형태)
   - 각 카드: 파트 번호, 파일 크기, 재생 시간 범위
- **저장 완료 경로 표시** (클릭 시 폴더 열기)
- 저장된 폴더 열기 버튼
- "새 파일 처리" 버튼

## 5. API 설계

### 5.1 엔드포인트

| Method | Path | 설명 |
|--------|------|------|
| `GET` | `/` | 메인 페이지 서빙 |
| `POST` | `/api/upload` | 동영상 파일 업로드 |
| `POST` | `/api/split` | 동영상 분할 실행 |
| `GET` | `/api/status/{task_id}` | 분할 진행 상태 조회 (폴링) |
| `POST` | `/api/validate-path` | 저장 경로 유효성 검증 |
| `POST` | `/api/browse-folder` | 네이티브 폴더 선택 다이얼로그 (tkinter) |
| `GET` | `/api/default-path` | 기본 저장 경로 조회 |
| `POST` | `/api/open-folder` | 탐색기에서 폴더 열기 |
| `DELETE` | `/api/cleanup/{task_id}` | 임시 파일 정리 |

## 6. 비기능 요구사항

### 6.1 성능
| 항목 | 목표 |
|------|------|
| 분할 속도 | `-c copy` 사용 시 원본 크기 대비 10% 이내 시간 |
| 동시 처리 | 단일 사용자 기준 (로컬 도구) |
| 메모리 | FFmpeg 스트림 처리로 메모리 효율적 처리 |

### 6.2 안정성
| 항목 | 목표 |

| 에러 처리 | FFmpeg 오류 시 사용자에게 명확한 메시지 표시 |
| 파일 정리 | 처리 완료 후 임시 파일 자동 정리 (설정 가능) |
| 유효성 검사 | 지원하지 않는 형식 업로드 시 사전 차단 |

### 6.3 사용성
| 항목 | 목표 |
|------|------|
| 설치 | Python + FFmpeg만 필요, npm 불필요 |
| 실행 | `python server.py` 한 줄로 시작 |
| 브라우저 | Chrome, Edge, Firefox 최신 버전 지원 |

## 8. 제약 사항 및 고려 사항

> [!WARNING]
> ### FFmpeg 키프레임 분할 주의사항
> `-c copy` (무손실 복사) 모드로 분할 시, 정확한 용량 기준 분할이 어려울 수 있습니다. 키프레임(I-frame) 단위로만 분할 가능하므로, 목표 용량은 **근사값**으로 처리됩니다. 분할된 각 파트가 목표 용량을 약간 초과하거나 미달할 수 있습니다.

> [!NOTE]
> ### 로컬 전용 도구
> 이 프로젝트는 로컬 환경에서 실행되는 도구입니다. 클라우드 배포나 다중 사용자 지원은 현재 범위에 포함되지 않습니다.

## 9. 성공 기준

| 기준 | 측정 방법 |
|------|-----------| 
| 분할 정확성 | 모든 분할 파트가 목표 용량 이하 |
| 영상 품질 | 분할 후 영상/음성 품질 저하 없음 |
| 연속 재생 | 분할된 파트를 순서대로 재생 시 원본과 동일한 내용 |
| 사용 편의성 | 3클릭 이내에 분할 완료 (업로드 → 설정 → 시작) |
| 안정성 | 1GB 이상 파일도 오류 없이 처리 |
